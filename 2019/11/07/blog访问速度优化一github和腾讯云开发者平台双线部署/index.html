blog访问速度优化一github和腾讯云开发者平台双线部署
发表于 2019-11-07| 更新于 2019-11-07|hexo加速优化
字数总计: 754|阅读时长: 2 分钟| 阅读量:

概述

前两天好不容易搭建的博客，怀揣激动的心情让朋友访问时，被吐槽加载好半天都出不来。难顶，以前放自己服务器上的小项目也从来没有关注类似访问速度优化的问题，那就来搞一搞吧。

分析

​ 普通的hexo+github的模式，在国内访问速度较慢，而且github.io不会被百度收录 ，所以在将blog站点部署到国内的站点是一个不错的选择。

所以我们要做的就是用hexo+github+腾讯云开发者平台（我不知道Coding和他具体发生了些什么，用Coding的时候已经变为腾讯开发者平台了，心累），将本地的blog部署到两个平台，最后通过域名解析，海外路线指向到github的地址( username.github.io )，国内默认指向腾讯云开发者平台( username.coding.me )。

Do it

首先默认你已经在github成功部署了你的blog。然后你必须要有一个腾讯云开发者平台账号。
